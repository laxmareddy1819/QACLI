Yes! You already have two API tools available through natural language chat in the qabot REPL:

1. api_request — Send any HTTP request
Just ask naturally in the REPL:

"Send a GET request to https://petstore3.swagger.io/api/v3/pet/1"
"POST to https://api.example.com/users with body {"name": "John", "email": "john@test.com"}"
"Call the pet store API to create a new pet with name 'Buddy'"

The LLM will use the api_request tool — supports all HTTP methods, headers, auth (bearer/basic), query params, JSON/text/form/GraphQL bodies, timeouts, and redirects.

2. api_validate_schema — Validate responses against JSON Schema
"Validate this response against the User schema in my OpenAPI spec"

It can validate against inline JSON schemas or extract component schemas from OpenAPI/Swagger spec files.

Plus the LLM can also:
Use read_file to read your OpenAPI/Swagger spec files
Use write_file / edit_file to create test files
Use run_command to execute tests
Chain multiple API calls with variable extraction
So from the CLI, you can say things like:

"Read my swagger.json and test all the pet endpoints"
"Create a Playwright API test for the POST /pet and GET /pet/{id} endpoints"

The UI dashboard (/buildUI) gives you the visual collection builder, environment management, and the rich Generate Test dialog we just built — but the core API testing capabilities are already available through natural language in the REPL. No additional changes needed!
