Context:

qabot already has an internal self-healing system (src/healing/) with SelfHealer (4-strategy chain), ElementFingerprinter (weighted attribute comparison), and HealingStore (SQLite persistence). However, this only works within qabot's own Playwright browser sessions. The user wants to expand this into a universal auto-healing feature that can inject self-healing capabilities into any test automation framework regardless of tools and languages (Playwright, Cypress, Selenium, WebdriverIO, Robot Framework, Appium, Cucumber — in JS/TS/Java/Python/C#/Ruby).
Feasibility: YES — fully feasible. qabot has 80% of the foundation already built. The missing piece is a framework-agnostic healing bridge and native-language code generators.

Architecture: The Healing HTTP Bridge
External frameworks (Java Selenium, Python pytest, C# NUnit, etc.) cannot call qabot's Node.js healing engine directly. The solution is a Healing API on qabot's Express server that generated framework libraries call over HTTP.

Why HTTP bridge? Java/Python/C# can't call Node.js in-process. HTTP is universal, adds <50ms latency on localhost, and the generated libraries handle connection failures gracefully (log warning, proceed without healing).

Context
The self-healing system has two implementations with different strategy coverage:

SelfHealer (src/healing/healer.ts) — used internally with a live browser — runs 5 strategies in sequence: fingerprint → similarSelector → textMatch → positionMatch → ancestorSearch, plus aiHealing as fallback. Each returns null on failure, loop continues. Stops on first success.
Universal Healing API (POST /api/heal/resolve) — used by ALL 10 adapter frameworks via HTTP — generates candidates from stored fingerprint attributes only (a single "fingerprint" pass). AI healing only triggers when zero deterministic candidates exist. No multi-strategy fallback.

The Complete 6-Strategy Chain (Goal)

```
## Part 3: Client-Side — Multi-Candidate Retry + AI + Vision Fallback

### Complete client-side pattern (pseudocode):
```
// Phase 1: Deterministic candidates
healResult = POST /api/heal/resolve(brokenSelector, ...)
candidateList = [healResult.selector] + healResult.candidates (deduplicated)

for each candidate in candidateList:
  try:
    result = tryAction(candidate)
    report(success, candidate)
    return result
  catch:
    continue

// Phase 2: AI text-based healing (only if Phase 1 exhausted)
aiResult = POST /api/heal/resolve(brokenSelector, requestAI: true)
if aiResult.healed:
  try:
    result = tryAction(aiResult.selector)
    report(success, aiResult.selector, 'aiHealing')
    return result
  catch:
    // continue to Phase 3

// Phase 3: Vision healing (screenshot + multimodal LLM)
screenshot = captureScreenshot()   // Playwright: page.screenshot()
visionResult = POST /api/heal/vision(brokenSelector, screenshot, ...)
if visionResult.healed:
  try:
    result = tryAction(visionResult.selector)
    report(success, visionResult.selector, 'visionHealing')
    return result
  catch:
    report(failure)

throw originalError
