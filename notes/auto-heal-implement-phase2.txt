Implementation Order
Step 1: DOM Analyzer (no dependencies — build first)
New file: src/healing/dom-analyzer.ts

DOMAnalyzer class with three capabilities:

extractContext(domSnapshot, fingerprint, scope) — extracts relevant HTML section around the element's expected position for LLM context scoping. section mode caps at ~2KB, minimal ~500 bytes, full ~8KB
validateSelector(domSnapshot, selector) — lightweight regex-based check if selector's target attributes exist in the HTML, returns {valid, matchCount, stabilityScore}
scoreStability(selector) — ranks selectors: [data-testid=...]=1.0, #id=0.9, [aria-label=...]=0.8, [name=...]=0.7, .class=0.5, tagName=0.3
rankCandidates(domSnapshot, candidates) — sorts candidates by combined validity + stability

Step 2: AI Healer (depends on Step 1)
New file: src/healing/ai-healer.ts

AIHealer class using getRouter() from src/llm/index.js for one-shot router.complete() calls
heal(originalSelector, fingerprint, domSnapshot?, snapshotScope?) → AIHealResult | null
System prompt: expert at fixing broken CSS selectors, respond with JSON {selector, confidence, reasoning}
Uses DOMAnalyzer.extractContext() to scope DOM context before sending to LLM
Temperature: 0.2, maxTokens: 256 — deterministic, fast
Graceful fallback: dynamic import() of router, try/catch everywhere, returns null on any failure
parseResponse() handles clean JSON, markdown-wrapped JSON, and malformed responses
