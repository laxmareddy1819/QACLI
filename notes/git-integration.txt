Context
The qabot UI dashboard has zero git integration. Test results capture file paths, errors, and stack traces but have no connection to version control. When a test fails, there's no way to know who last modified the file, which commit introduced the failure, or how the file has evolved. This feature adds 6 git-powered capabilities across 3 phases.

Step 2: Create src/ui/services/git-service.ts (NEW)
Core git operations service using simple-git. Constructor takes projectPath.

isAvailable(): boolean — checks git rev-parse --is-inside-work-tree
getStatus() — returns { branch, isClean, lastCommit: { sha, shortSha, author, email, message, timestamp }, uncommittedChanges[], aheadBehind }
blame(filePath) — parses git blame --porcelain into GitBlameEntry[] + summary (lastModifiedBy, lastModifiedAt, lastCommitSha, lastCommitMessage)
getFileHistory(filePath, limit=20) — git log --follow --stat → commit array
getRecentCommits(limit=20) — git log with --name-only → commits with changed files
getCommitDiff(sha) — git diff sha~1..sha → file diffs
getCommitsBetween(fromSha, toSha) — git log fromSha..toSha with --name-only
getChurnScore(filePath, days=30) — count modifications in time window

All methods are async, use simpleGit(projectPath), and have try/catch returning null/empty on failure.

Phase 2: Commit Correlation + Auto-Assignment
Step 13: Add correlation methods to GitService
Add to src/ui/services/git-service.ts:
typescriptasync correlateFailures(currentRun: StoredRun, previousRun: StoredRun | null): Promise<CommitCorrelation[]>
Logic:

Get commits between previousRun.gitCommitSha and currentRun.gitCommitSha
For each new failure (test failed in current but passed in previous):

Check if any commit modified the test file → confidence high
Check if any commit modified a file in the stack trace → confidence medium
Otherwise temporal correlation → confidence low


Return array of CommitCorrelation objects

Logic:

Extract file:line from test.stackTrace using regex
git blame the failing line → primary owner
git blame the test file → secondary owner (last modifier)
Return { suggestedOwner, alternativeOwners, confidence, reason }
