Context
This guide covers step-by-step testing of the entire Universal Auto-Healing feature (Feature 5) across all 3 phases:

Phase 1: Core healing engine, 6-strategy chain, 3 adapters, HTTP bridge, HealingStore, CLI tools
Phase 2: AI Healer, DOM Analyzer, 3 more adapters (Cypress/WDIO/C#), enhanced React dashboard
Phase 3: Settings UI, Fingerprint Browser, Event Detail Panel, WebSocket real-time updates, error handling


Prerequisites
bash# 1. Build everything
pnpm run build:ui && pnpm build

# 2. Start the qabot UI server (port 3700)
node bin/qabot.js /buildUI

# 3. Open browser
# Navigate to http://localhost:3700
```

> **Note**: The healing SQLite database is stored at `~/.qabot/healing.db`. If you want a clean test, delete this file before starting.
```
